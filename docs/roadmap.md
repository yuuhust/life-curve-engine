# Life Curve Engine Roadmap

## 1. 当前 V1.9 状态

V1.9 已完成从记录到闭环验证的最小链路：

```text
行为输入 → 规则引擎 → 曲线/解释 → 建议 → 计划 → 闭环验证
```

当前重点是稳定内核、明确参数、让系统可解释。

## 2. V1.x 后续建议

### V1.10：闭环验证增强

- 给计划增加创建时间和更新时间。
- 验证时使用计划开始后的记录，而不是最近 3 天窗口。
- 保存每次验证结果，形成轻量执行历史。

### V1.11：解释链增强

- 输出更明确的路径：行为 → 实体 → 中间因子 → 曲线。
- 将解释模板配置化。
- 给建议附带“为什么是这个建议”的结构化原因。

### V1.12：最小 AI 润色入口

- 输入：`ReviewSummary`、`AdviceSummary`、`PlanValidationSummary`。
- 输出：自然语言复盘摘要和建议润色。
- 不让 AI 改规则、权重、分数。

## 3. V2 候选方向

最适合进入 V2 的方向：个性化权重与阶段模式。

原因：

- 当前系统已有稳定内核和完整闭环。
- 不同用户阶段差异很大，例如考研、求职、实习、创业。
- 个性化权重可以复用现有 `entityWeights`、`intermediateWeights` 和规则表。

V2 可以考虑：

- 阶段模式：学业优先、求职优先、项目优先、健康修复。
- 用户可调权重。
- 历史重算和权重版本管理。
- 更强的周/月复盘。

## 4. 暂缓事项

- 复杂多页面系统。
- 复杂任务管理。
- 自动抓取外部数据。
- 让 AI 直接参与评分。
- 多用户和社交功能。
